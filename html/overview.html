Overview

The World Wide Web, aka the WWW , has undergone many changes since it’s inception a few decades ago. HTML5 is the latest iteration of the primary markup language that makes webpages webpages. If you were indoctrinated into the web with another version of HTML, or XHTML, have no fear, your knowledge is not considered obsolete. If you have only heard about HTMl in passing, don’t worry, we’ll walk you through the basics.

A few questions you may have at this point may be:

If HTML5 is the lastest version, does that mean that HTML4 or XHTML don't work?

Technically, yes previous versions will work. Which is why we still see remnants of the nineties and its, shall we say, distinctly antiquated webpages emerge from the dark every now and then. If it didn’t work, then we couldn’t see it (unfortunately).

In all seriousness, we can see previous versions of html because, unless the tags are deprecated, HTML5 enfolds all iterations within its specification.

What makes HTML5 different if old versions are allowed?

HTML5 is relatively new, so some browsers may not support some of its features. Developers have been using workarounds since more than one browser appeared on the seen, so this is nothing new. HTML5 is expected to fully supported by all browsers by 2022. In the meantime, to see which HTML5 features are supported by which browsers, see here. (html5test).

As you'll soon see, the language that makes up HTML5 has not changed significantly over over the year, it's only been refined to 1) integrate features that previously required added plug-ins into the webpage and 2) make it easier for web designers to design.

So, what does HTML(5) actually do?

HyperText Markup Language, or HTML, provides the infrastructure, that is to say,building blocks, for the content you want to display...

In short, with HTML you can:

  • Create infrastructure for your content in various ways
  • Link to other websites or sections within the same website or webpage
  • Embed various types of "non-text" content in your webpage such as, images, audio, etc"
  • Create interactive forms
  • Embed scripts which can effect the behavior of the content (i.e. make it do cool things)

What you can't and/or shouldn't do with HTML:

  • Make things pretty
