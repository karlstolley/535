Tables

TTables are used for tabular dataables are a series of rows and columns that create cells in which data lives.

Tables are built using essentially three tag elements: <table> </table>, <tr> </tr>, and <td></td>.

To create a table, you begin with the <table> </table> tags. Anything within these tags are part of the table. Tables are built row by row using the <tr> and </tr> tags. Creating tables is another example of where hierarchal nesting is important. Let's take a look at our sample project. 

    	<table>
	<tr>
	<th>Education</th>
	<th>Experience</th>
	<th>Skills</th>
	</tr>
	
	<tr>
	<td>taciti</td>
	<td>litora</td>
	<td>conubia</td>
	</tr>
	
	</table>

As you can see above it all begins with the opening <table> tag. The <table><table> tags indicate the top row heading, which, by default, is bold (can be changed with CSS). The <table> <table> tags contain the table cell data. Our example has a table. Each individual data cell are housed within the <td> </td> tags.s

The HTML above gives you this

Education Experience Skills
taciti litora conubia

If you wanted to add more rows, you simple continue with the same structure within the table element. Just make sure you enclose each row of data with the <tr> and </tr> tags.

To set the . If you are creating a small table, th is good. but for large tables or perhaps to make it easier on yourself, use thead. In each row element, each td element is a column, in the order in which it is presented. Number of columns is determined by the number of td elements. If we were to add an additional td element, we would be therefore be also adding a column.

thead, tbody, and tfoot

Although a table could be built with just the 3 tags, HTML5 also provides three tags that break the table into natural sections for styling purposes [aside: use CSS]. The <thead>and </thead> tags enclose the data in the first section to define the heading row of the table. Similarly, the <tbody>and </tbody> and the <tfoot>and </tfoot> tags enlose and define the body and footer data respectively.

Unlike the th tag, when the thead tag is used, there is no need to specify the order when you are writing the html. the browser will recognize the thead should be placed on top of the tbody; similarly with the tfoot tag, there is no need to make this come at the end of the rows of td. For example, 
    	<table>
	<thead>
	<td>Education</td>
	<td>Experience</td>
	<td>Skills</td>
	</thead>
	
	<tr>
	<td>taciti</td>
	<td>litora</td>
	<td>conubia</td>
	</tr>
	
	</table>
