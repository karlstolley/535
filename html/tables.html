Tables are used to present data and other similar types of information in a grid. Tables are are a series of rows and columns that create cells that house each individual piece of information. Tables should not be used for layouts, instead, you should use CSS.

Tables

Tables are built using essentially three tag elements: <table> </table>, <tr> </tr>, and <td></td>.

To create a table, begin with the <table> </table> tags. Anything within these tags are part of the table. Tables are built row by row using the <tr> and </tr> tags. Let's take a look at a part of our sample project. 

    	<table>
	<tr>
	<th>Education</th>
	<th>Experience</th>
	<th>Skills</th>
	</tr>
	
	<tr>
	<td>taciti</td>
	<td>litora</td>
	<td>conubia</td>
	</tr>
	
	</table>

As you can see above, it all begins with the opening <table> tag. The next tag is the opening <tr> tag, which indicates the first row in the able. It is followed by three table header elements, which use the table header tags<th> and </th>. The table header elements indicate that the first row, the row we've just created with the <tr> tag, is a header row. By default, all content within the <th> and </th> tags is bold. The row, with its table header elements, end with the closing </tr> tag.

The next row, as indicated by the next opening <tr> tag, contains the cell data. The <td> and </td> tags contain the table cell data. These cells are not bolded and are displayed in the default font.

Change style and fonts with CSS.

The number of columns is determined by the number of table data elements, for example <td>Education<td>. If we were to add an additional table data element, we would be also adding a column. When adding columns, be sure to make sue you incude the same number table data elements for all table rows.

If you wanted to add more rows, you simple continue with the same structure within the table element. Just make sure you enclose each row of data with the <tr> and </tr> tags.

After you have finished added the rows of data you want to include in your table, close the table with the <table>

taciti litora conubia

thead, tbody, and tfoot

Although a table could be built with just the three tags (<table>, <tr>, and <td>), HTML5 also provides three tags that break the table into natural sections for styling purposes. The <thead>and </thead> tags enclose the data in the first section to define the heading row of the table. Similarly, the <tbody>and </tbody> and the <tfoot>and </tfoot> tags enlose and define the body and footer data respectively.

Unlike the <th> tag, when the <thead>tag is used, there is no need to make sure that the header row is first when you are writing the html. The browser will recognize the thead element should be placed on before the tbody element. Similarly, with the tfoot tag, there is no need to make this come at the end of the rows of table data. For example, 

    	<table>
	<thead>
	<td>Education</td>
	<td>Experience</td>
	<td>Skills</td>
	</thead>
	
	<tr>
	<td>taciti</td>
	<td>litora</td>
	<td>conubia</td>
	</tr>
	
	</table>

With the code><thead>, <tbody>, and </tfoot> tags, it is possible to change the orders of these elements without affecting the page.
