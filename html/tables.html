Tables

Tables are used to present data and other similar types of information in a grid. Tables are are a series of rows and columns that create cells that house each individual piece of information. Tables should not be used for layouts, instead, you should use CSS.

Tables are built using essentially three tag elements: <table> </table>, <tr> </tr>, and <td></td>.

To create a table, use the <table> tag to start the table and the</table> tag to end the table. Anything within these tags are part of the table.

Tables are built row by row using the <tr> and </tr> tags. Let's take a look at a part of our sample project. 

    	<table>
	<tr>
	<th>Education</th>
	<th>Experience</th>
	<th>Skills</th>
	</tr>
	
	<tr>
	<td>taciti</td>
	<td>litora</td>
	<td>conubia</td>
	</tr>
	
	</table>

As you can see above, the table begins with the opening <table> tag. The next tag is the opening <tr> tag, which indicates the first row in the able. It is followed by three table header elements, which use the table header tags <th> and </th>. The table header elements indicate that the first row, the row we've just created with the <tr> tag, is a header row. By default, all content within the <th> and </th> tags is bold. The row, with its table header elements, end with the closing </tr> tag.

The next row, as indicated by the next opening <tr> tag, contains the cell data. The <td> and </td> tags contain the table cell data. These cells are not bolded and are displayed in the default font.

Change style and fonts with CSS.

The number of columns is determined by the number of table data elements, for example <td>Education<td> is one data element. If we were to add an additional table data element, we would be also adding a column. When adding columns, be sure to make sue you incude the same number table data elements for all table rows. Otherwise, your table may end up like a series of uneven stairs, if you're lucky.

If you want to add more rows, you simple continue with the same structure within the table element: <tr> tag followed by as many data elements (columns) as you need, then the row is closed with the </tr> tag.

After you have finished adding your rows of content, close the table with the <table>

The above HTML will give you this...

Education Experience Skills
taciti litora conubia

The structure above is fine for creating small tables that do not contain a lot of content, but sometimes they can become too unwieldy to make it worth it. For larger tables, or, perhaps, if you want to add styling (with CSS) to different parts of the table to make them stand out it is easier to break the table down into different sections: the table header, table body, and table footer.

thead, tbody, and tfoot

Although a table could be built with just the three tags (<table>, <tr>, and <td>), HTML5 also provides three tags that break the table into natural sections for styling purposes. The <thead>and </thead> tags enclose the data in the first section to define the heading row of the table. Similarly, the <tbody>and </tbody> and the <tfoot>and </tfoot> tags enlose and define the body and footer data respectively.

Unlike the <th> tag, when the <thead>tag is used, there is no need to make sure that the header row is first when you are writing the html. The browser will recognize the <thead> element should be placed on before the tbody element. This is also the case for the <tfoot> tag; there is no need to write the <tfoot> at the very end of the rows of data in the body of the table. For example, 

    	<table>
	
	<tfoot>
		<td>Footer 1</td>
		<td>Footer 2</td>
		<td>Footer 3lt;/td>
	</tfoot>
	
	<thead>
		<td>Education</td>
		<td>Experience</td>
		<td>Skills</td>
	</thead>
	
	<tbody>
	<tr>
		<td>taciti</td>
		<td>litora</td>
		<td>conubia</td>
	<tbody>
	
	
	</table>
