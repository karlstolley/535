There is clearly a lot that went into such a simple looking document, but before we go on, let's define some terminology.

Tags

A tag is the simplest unit of HTML, and basically defines the language. Browsers read tags for instructions on how to display the content in a webpage. A browser recognizes an HTML tag by the < and > that enclose each particular instruction. Acting as bookends, the < and > lets the browser know when the tag begins (<) and ends (>). Let's take another look at the project code above for an example. On the second line, we see the <html> tag. The <html> lets the browser know that the what follows is a webpage.

But wait, there's more. Just as the tag needs both an < and an >, to define it, the element that the tag modifies needs something to let the browser know when the instruction ends, the </>. So think of each tag as a set of bookends, one at the beginning, called a start or opening tag, < > and one at the end, a end or closing tag < / >. [aside: some browsers allow for a looser format and may not require an ending tag but, best practices require both start and end tags] There are three exceptions to this "bookend" rule: one we discussed earlier - <!DOCTYPE html>which tells the browser which version of html it is using; </br>, which we'll discuss in the next sections.

As we return to our example, we can see that the <html> start tag at the beginning, has a closing tag, </html> all the way at the very end of the page. In fact, it's the last tag, which makes sense because it is basically telling the browser that there is nothing more to the webpage.

Attributes

An attribute is part of the tag that gives additional instruction to the browser. It is always found in the opening tag, after the tag name (in this case html) and before the end >. An attribute consists of a name and a value. Let's look at the html example again. In the opening <html> tag, there is the attribute lang="en". Here the name is lang which defines the language for this particular html document, and the value en, which tells you the language is English. The value is always enclosed in quotes " ".

You will get much more involved with attributes as you learn about CSS.

Elements

An element consists of a set of opening and closing tags and the information it encloses, including the attributes and values. For example, in the sample project, we see that there are more than one meta elements, which we will learning more about in the next section, each of which begin with the opening meta tag </meta> has some information then end with the closing tag </meta>. The tags by themselves <meta> and </meta>are simply tags.

Sometimes you may see, the term element used interchangeably with tag. Let's take for example the element <!DOCTYPE html>. It is an element that stands by itself. Let's find another example. below that is <html>. but not simply <html>, it also includes the tag and value, lang="en" where the tag is lang and the value is en.
