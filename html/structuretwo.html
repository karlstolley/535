Body: Structure Part II

Tags used within the body of a webpage continue with other structural tags that help define smaller parts of the section, article, div, and footer sections

More Tags Used Within the Body

Header

The header begins and indicates that the information enclosed in the <header> and </header> is the header. Content within the header could be as simple as the title of the section, but it could also include supplementary information. Although there should only be one header per section or article, since there could be multiple sections and articles in page, there could be more than one header per page. Each header element must contain at least one heading tag (see below).

H1, H2, H3, H4, H5, and H6

The heading tags are used to set apart certain text from the rest of content, usually in size and font weight. By default, and generally by design, the <h1>and </h1> tags indicate the most important information and when used are a much larger size and weight. The style of the tags continue to decrease in size and font weight from the <h2> tag to the <h6> tag. For example, on this webpage, the title of the tutorial, HTML5, uses the <h1> tag, then the title of the page Body: Structure Part I uses the <h2> tag, then the title of the section More Tags Used Within the Body uses the <h2>tags, and the title of each paragraph uses the <h4> tags. Always be sure to close the tag with the appropriate matching end tag.

Paragraph

The <p> tag is probably the most common tag you will see in a document, simply because it defines the portion of text that should be displayed as a paragraph. It is possible for a very simple page to be composed simply of <p> </p> elements, however, it is very limited and it doesn't tell you much about the information enclosed. Also, when we start to style the page using CSS, other tags, such as section and article help better define parts of the webpage.

Unordered and Ordered Lists

Lists are used with surprising frequency when building webpages. The most common use is with navigation links but there are also other instances, such as the assumptions and learning objectives at the start of this tutorial. Lists help display information in an attractive and easy to read format, without the need for styling (though it's always looks better with a little style, see CSS).

There are two different tags for lists, depending on how you would like the information displayed: <ul> and <ol>. The <ul> tag is used for unordered lists, when you do not want do put the information in any particular number order. Depending on your styling, or, if defined in styles, the browser default, a simple round bullet point precedes each item in the list.

The <ol>, or ordered list, tag is used when you want the information to be displayed in order and using a number. The browser will automatically list the items in order 1, 2, 3, 4, etc, based on how you list it in the code.

In both cases, for each individual listed item you need to use the opening tag <li> and the closing tag </li>. If, for example, we wanted to create an ordered list, we would start with the opening <ol> tag. Then we would list each item in the list within the <li> and </li> tags. Once we are done, we would close the list with the end <ol> tag.

Aside

The aside tags <aside> </aside> are used for pieces of content that relate to an article or webpage's content but could stand on its own or may not be necessary to understand the rest of the content. Asides could be presented in various ways. They are used much like sidebars are used in print documents, though in websites, they don't necessarily have to be sidebars. Another example would be of a pull quote from a long article that highlights the topic, or glossary of terminology, or related links for further information. In this guide, you see examples of how they are used to present extra information.
