The Head

<!DOCTYPE html>
<html lang="en">
<head>
<meta charset="utf-8">
 <title>HTML5 Sample</title>

  <!--Brief page description:-->
  <meta name="description" content="">
  <!--Name(s) of primary authors-->
 <meta name="author" content="">

  <!--Styles-->
<!-- <link href="/assets/css/bootstrap.css" rel="stylesheet"> -->

</head>

Webpages consist of two major elements: the head and the body. We'll deal with the head now then go on to learning more about the body.

Head

As you can see in the project HTML, the tag comes right after the opening <html> tag is the <head> start tag. Look a little below, you can find the <head> end tag. The section enclosed in these tags is called the HEAD of the document and contains important information for the web designer and the browser to know but does not necessarily get displayed for the public. Here, you include META information (see below) as well as links to stylesheets and scripts (though scripts could be placed below the body as well)

For more information about stylesheet and scripts see the CSS and Javascript chapters
. There is no particular order in which the elements should be presented; however, the only information that is necessary to include in the HEAD is the title.

Tags and Elements Within the Head

Title

The title tags <title> and </title> let the browser know what the name or title of document is. Depending on your browser, the title is typically displayed at the in the title bar of browser window, or if you are using browser tabs, in the tab. It helps orient the user to the name of the page. It also appears as the name of the document in the user's favorites listing if the user chooses to bookmark the page.

Base

The base element identifies the base URL of the document, that is, its the main website URL, in order to make it easier to use relative webpages for hyperlinks. As we will see more of when we go to the link section of this chapter.

Link

The <link> and </link> tags are used to link to external resources that affect the behavior or look of the webpage. One common external resource is the CSS stylesheet. An attribute is needed to let the browser know what type of link it is. For stylesheets, the name is ref and the value is "stylesheet". For each additional stylesheet, a new link element is needed. We need to use the href attribute to identify where the stylesheet is housed. The value of the attribute will be the destination. Depending on the type of link, the attribute is handled in different ways, as we will more about in Links and Images.

Script

Like the link tags, the script tags <script>and </script> are used to define the external resources that affect the behavior of the webpage. However, unlike stylesheets which affect the look of the page, the external resources defined by the script tags are scripts which affect client-side interaction scripts, usually Javascript or Jquery.

For the script tags, the src attribute is used to identify the external sources.

Meta

Meta information is any extra information about the particular webpage that describes the document or provides helpful explanations for future reference. Meta information is enclosed in the <meta> and </meta> tags. Let's take a look at the meta information for our project. 

<!DOCTYPE html>
<html lang="en">
<head>
<meta charset="utf-8">
 <title>HTML5 Sample</title>

  <!--Brief page description:-->
  <meta name="description" content="">
  <!--Name(s) of primary authors-->
 <meta name="author" content="">

  <!--Styles-->
<!-- <link href="/assets/css/bootstrap.css" rel="stylesheet"> -->

</head>

Here, we've included meta information options for name and for author. Other meta information we can include would be search tags for browsers using the name keyword and the search words would be in the value content. It helps search engines find information faster about the page and helps you relate topics that may not be mentioned explicitly in the body of your page.

Comments

So what if there is information you want to include for yourself or another developer, and not for the browser? That where comments come in. Comments are extremely useful to the content creator because it helps with reminders about certain code especially when a significant a amount of time lapses in between uses. In our sample project above, we've used comments to let us know how we are defining certain types of meta information. We would also include it to help distinguish types of styles and scripts from one another. To create a comment in html, simple add a <, an exclamation point ! and two dashes -- to form a <!--. Place your comment, then enclose it with two dashes -- and a >. Like all elements in html,comments must be enclosed; in this case, with a start <!-- and ending -->.

Comments are not confined to the head of an html document; they can also be used in the body.
