Links and Images

Links, also called hyperlinks, allow you to connect to other parts of a website and external websites. Images provide illustrative imagery to your content.

Links

Now, let's learn about what makes a webpage truly a webpage—links. If not for links, a webpage would be a static document isolated from the rest of the internet, nearly impossible to reach and just as impossible to leave. Links are what makes the world wide web, the world wide web.

Here, we will learn about the two types of links: internal links and external links. But before we learn more about what makes each of these types of links unique, let's take a closer look at the link tag and its individual components.

Links are formed with the start tag <a> and end tag </a>.

The origin anchor, also known as the source anchor, is where the link begins and is identified by the presence of the opening <a> tag. The target anchor is identified within the opening tag with the href attribute. The value of the attribute will be the link destination. Depending on the type of link—local page or external—the attribute is handled in different ways, as we will soon see below.

For a refresher on attributes, visit the Tags, Attributes, and Elements section of this tutorial.

Basically, the element structure for links is <a href="LINK DESTINATION"> WHAT VIEWER SEES AS A HYPERLINK</a>.

Be sure not to confuse the <a> tag with the <link> tag; <a> tags only appear in the body, and <link> tags only appear in the head of the document

External Links

We'll start with the simplest types of links—the external link.

To insert a link in your content to an external page is use the <a> and </a>tags and insert the complete web address as the destination value. For example, if we wanted to provide a link from this page to the IIT home page, we would simple type <a href="www.iit.edu"> IIT </a>, which appears to the viewer as IIT.

Internal Links

Internal page links connect to other webpages within the same website. For example, the various sections of this tutorial are housed on different pages within the same website- DigitalContentCreatorsResource.com. In order to create links to internal pages, you must know where the content for your pages resides. For example, the pages for this HTML5 tutorial are housed together in the same folder. If I wanted to link to another page within this tutorial, let's say the Overview, then I would list the name of the html document as the destination value in the attribute. Let's see the this in action: <a href="overview.html"> Overview </a>. As we can see, the anchor tag <a> contains the value as the name of the document. The viewer would only see Overview.

Images

Images are part of the same tutorial as the links because they share the same concept only with different tags. The opening <img>tag contains the src attribute whose value identifies the image that should appear. Here, is where the idea closely follows the way an internal link is made. The value of the src attribute is usually within the same website that the webpage you are creating is in, usually within a folder named Images. Let's take a look below at a portion of our sample project for an example. 

    <h2>My Future</h2>
<img src="images/oceanview.jpg" alt="my future image">

As we can see above, the <img> tag is used to indicated that an image is being inserted into the document. The value of the src attribute is the location of the image, which is in the Images folder.

Also above, we can see another attribute, the alt. The value for the alt attribute is the alternate text that would appear if for some reason the image did not load.

Remember, when we learned that there were several tags that did not require end tags. The <img> tag is one of them.
