Positioning Elements

Positioning is such an important part of CSS that it deserves its own section. Here it is.

Document Flow

The normal document flow process of loading a web page can be briefly explained as this:

  1. When the browser loads your web document, it reads from top to bottom, left to right.
  2. The elements are initially laid out on the page in that order. If you have an <H3> tag listed before an <H1> tag, the <H3> tag will be displayed first.
  3. Once the elements are laid out out in the browser, your CSS rules will be applied to the document. Here is were you can say that elements 1-2-3 should display as 3-1-2 if you like.
  4. This usually happens far too quickly to see that elements get repositioned on the screen.

The Box Model

Learning about positioning in CSS requires a discussion about the box model. Every HTML element is considered to be a "box." The browser will see boxes (or: rectangles) of content to be display, whether they're paragraphs, headings, images or videos: this, in short, is the box model. Some boxes are displayed as block-level elements, and some as inline elements. Regardless, the following properties will affect how each box appears within the browser:

  • width
  • height
  • padding (amount of space between the element and its border)
  • margin (amount of space between the element's border and its adjacent elements)
  • position offsets: top, bottom, right, left

all of these things will affect the box model. If an element has the following properties:

  width: 30%;
  top: 20%;
  left: 20px;
  padding: 20px;
  margin-bottom: 40px;
  border: 2px dashed red;
You could also write the margin property this way: margin: 0 0 40px 0. This is helpful if you want to specify different values for the top, right, bottom and left attributes but want to use only one line. Just learn the word "TRouBLe" to remember the correct order.

The Display Declaration

The display property because that also has an affect on where an item appears within its container. Some elements, by default, are displayed on a separate line. Other elements are display in the same line as the elements adjacent to them

  • Block-level elements take up an entire line in the browser, even if the element's width is narrower than its container. All header tags: h1, h2, etc. and paragraph tags are block-level elements by default.
  • Inline elements continue on in the same line. Think of thie term as "in the same line." The <em> tag (used for "emphasis" in a sentence) is an inline-level element by default.
  • You can override the default display style for any element by create a specific style rule.

Absolute

The absolute position declaration is just as obstinate as it sounds. It's for when you absolutely, positively, have to have something stuck in one damn place. Absolutely-positioned elements are removed from the document flow, meaning they will be placed within the browser according to their positioning statements and ignoring things like their containing elements and other elements that might also occupy that space. This is how you can layer one object over another: with the absolute position.

For example, if we can drastically change the location of our Box 4 item, once we apply absolute positioning:

      width: 100px;
      padding: 10px;
      text-align:center;
      top; 10px;
      left: 200px;

Fixed

The fixed position is similar to absolute, but it takes it a step further. Not only is the position attached to a particular location, it is attached to the location within the browser window. This means that the item will not scroll off the screen. It is primarily used for footers and annoying internet ads.

      width: 100px;
      padding: 10px;
      text-align:center;
      left: 40%;
      bottom: 5%;
      position:fixed;

Inherit

Position declarations are not generally passed down to child elements (unlike declarations like font-size and background-color). The inherit position was designed so that you could do this.

Our boxes are sitting inside of a div container. We can have the boxes inherit the properties of the div container if we change the position declataion:

  div {
    position: fixed;
    top: 200px;
  }

  .box {
    position: inherit;
  }
The .box statement is a class declaration that will be discussed in the next section.

The Float Declaration

Floats can give your site the consistent look and feel that once was only achieved by stuffing your text into a table. They are also the cause of great frustration when things aren't displaying as you expected.

The Float property accepts three values:

  • left: float the element to the left side of its containing element (whether its another element of simple the browser window) and flow other elements to its right.
  • right: the opposite: float to the right and flow other elements around to the left.
  • none: do not allow the element to float.

Floats allow elements to be removed from the normal document flow and positioned so that other elements can flow around them. One of their main uses is to flow text around an image or table, like so:

Hmmm, suddenly I'm hungry. Why is that? And on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on...

Here is the code:

<img src="apple.jpg" />
<p>Hmmm, suddenly I'm hungry. Why is that? And on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on, and on...</p>

And here is the style that was applied. Note that the paragraph tag has no styling applied to it, only the image tag.

img{
      width: 70px;
      float:right;
      margin: 0 0 0 20px;
 }

  div {

    float:left;
  }

  div + p {
    clear:both;

If you have a complex layout, you can assign only the left or right values to the clear property. However, you'll likely use the both value most often.

If you only take away two points from this section, let it be these two:

  • If you want text or other items to flow around your floated element, be sure to assign a width as well as a float. If the element is by default a block-level display, then your next element will appear below the float.
  • When you want to stop the float of elements, assign a clear declaration to the following elelemt.
