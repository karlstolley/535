Declarations

Declarations are like the verbs of CSS. They tell the browser how to change the element. You can change just about anything about an element.

here are some examples:

  • Size: 16px, 12em, large, x-small, 80%
  • Color: red, #E01B6A, rgb(92, 140, 41)
  • Position: top: 100px, position: absolute, float: right
  • Background: green, url(images/bg.jpg), -moz-linear-gradient(top, #1e5799 0%, #7db9e8 100%)[a gradient]

Declarations consist of two parts.

Property

This is what about the element should change: color, font size, background

Value

This is what about the element should change to: red, x-small, 100px.

The Equation

Selectors, declaration properties and values combine to make a complete CSS rule. The structure you need to use is this: 

selector {
<declaration property> : <declaration value> ;
}
                       the colon separates
                         the property from
                         the value
                                             the semicolon ends the declaration statement
braces enclose the statements
Technically, you don't need a semicolon on the last line of your CSS rule. The same goes for the braces: if you only have one declaration in your CSS rule, you don't need to use them. But it's a good practice to always include them. It costs next to nothing in effort, and your code won't break if you add additional lines later. 
p {
    width:        50%;
    padding:      8px 4px;
    text-align:   center;
    margin:       22px 0 22px 15%;
    color:        navy;
    font-size:    20px;
    background:   rgb(250, 201, 110);
    border:       4px solid #a14016;
}


  properties    values

There are several ways to select colors in CSS.
  1. Named colors. An example is the text color above set to "navy." You can specify colors by their names: red, purple, green, etc. The obvious disadvantage is that you limited to fewer than 200 color names and must rely on the browser's version of how that color should display.
  2. Hexadecimal colors. One is used in the text color example above. The foumla is a pound sign "#" followed by 6 numbers/letters. These are much more specific: you have access to more than 16 million colors using this method.
  3. RGB colors. The background color above uses the RGB color specification. It defines color by specifying the amount of Red, Green and Blue to apply to each pixel. With this method, you can also add a level of transparency to the color, such as rgba(250, 201, 110, .5) to apply a 50% alpha transparancy to the color.

Styles vs. Elements

A common mistake when learning CSS is thinking that style rules are connected to HTML elements: that an <H1> tag will always display more prominently larger than a <p> tag, for example. This is not true. Web browsers will apply some default styling if none is supplied, but that can always be overridden. Below you see an example of h1 and p tags with your browser's default style rules applied. Click the button to override those styles:

See how the elements are no longer displaying according to the browser's default style rules? Here are the rules that were applied:

p {
    font-size: 34px;
    font-variant: small-caps;
}
h1 {
    font-size: 12px;
    text-transform: lowercase;
}
