Stylesheets

Add a Stylesheet

Once you start writing styles, it's hard to stop. This means the head section of your document is going to start filling up with CSS style rules. It's much more efficient to write them in a separate document and load it when the page is opened. This is called an external stylesheet.

The reference to an external stylesheet looks like this:

 <head>
  <!-- other head elements go here -->
    <link href="../assets/style.css" rel="stylesheet" type="text/css" />
</head>
<body>

The elements of the above reference are:

  • The link tag. This tells the browser that an external file will be loaded.
  • The href parameter. The location and name of the file.
  • The rel and type parameters. These indicate that the document contains CSS style rules.
  • The /> tag means this is a self-closing element. There is no need for a </link> tag.

Since you have defined the type of file in the link tag, your external style sheet will simply be a list of CSS rules.

  /* main_style.css file */

  body {
    padding 20px;
  }
  h1 {
    font-size: 24px;
  }

Import a stylesheet

You can also break your stylesheet up into different sections and import it into one main external stylesheet.

  • The import must be the first line(s) in your main external stylesheet.
  • Best practice tip: start your to-be-imported filename with "_<filename>" to avoid confusion.

Here's a sample of a to-be-imported stylesheet:

  /* _colors_and_fonts.css */
  html {
    font-family: Helvetica, Arial, san-serif;
    color:green;
    background:beige;
    margin: 10px 20px;
  }

And here's the main stylesheet that will import the stub file:

  @import url('_colors_and_fonts.css') /* import external file */

  /* main body styles */
  h1 {
    font-size: 18px;
  }
  p {
    font-size: 12px;
  }
Importing multiple stylesheets into one file is preferable to using multiple link tags because the browser only has to load one file.

Define a media-specific stylesheet

If you don't define a specific media type, the browser will assume your stylesheet is for all media, but you can identify different stylesheets for:

  • displaying on a monitor screen
  • printing
  • displaying on a projector
  • displaying on assistive devices, such as screen readers or braille printers
  • and more

One common use is create a print stylesheet for your resume that removes elements like headers and footers that you wouldn't want to print out. That stylesheet might look like this:

  @import url('main_style.css') /* import external file */

  /* hide header and footer */
  header, footer {
    display:none;
  }
