Using Classes and IDs as Selectors

Think you're limited to using only HTML elements as selectors? Think again.

So far, we've only discussed creating style rules for HTML elements. But using classes and IDs to create style rules allows For example, if you create a style rule for the p paragraph tag, this will apply to all paragraphs in your document, and that might not be what you want. Classes and IDs are special selectors that can be applied to a subset of HTML elements

For example, the first paragraph in this section ("Think you're limited...") is a paragraph, but the style is different than the other paragraphs. That's because this paragraph has been assigned the class name of "lead."

  <p class="lead">
    Think you're limited to using only HTML elements as selectors? Think again.
  </p>

This allowed me to assign CSS rules that only apply to elements that have the class name of "lead".

Classes

Classes are used when you want to apply an action to one or more HTML elements. An "action" in this chapter refers to CSS style rules, but it could also refer to a JavaScript method. In the next example, we'll create a class to use on some text columns.

Instead, we can add a class name to our column paragraphs. Note that when you assign a class name to an HTML element, the structure used is <element class="<className>">

   <h1>Fascinating headline here...</h1>
   <div>
     <p class="column">Column 1: text, text, text, text, text, text, text, text, text, text, text, text, text, text, text, text, text, text, text, text, text.</p>
   </div>
   <div>

   </div>
   <p>Here is a completely... </p>

Then, write a style rule for that class. When indicating a class name in a style rule, a different structure is used: .<classname>.

.column{
      font-size: 12pt;
      color: #0C17AB;
      font-weight:bold;
      font-family: "Helvetica", "Arial", sans-serif;
}

Multiple classes

You can also assign multiple classes to an element. This is helpful when you want to apply rules to different elements without having to repeat yourself (remember the DRY principle: don't Repeat Yourself).

Here's the code:

  <h1 class="center header">This is an H1 tag</h1>
  <h2 class="center header">This is an H2 tag</h2>
  <p class="center">Here is a paragraph of text.</p>
  <p class="center">Here is another paragraph of text.</p>
  <p class="center footer">This is the footer: &copy;2012</p>

Here are the CSS rules:

.center{
      text-align:center;
 }

 .header{
    color: #A913BD;
    font-weight:900;
  }

 .footer{
    color: #80248C;
    font-weight:900;
    font-size:10px;
    margin: 40px 0 0 0;
  }

IDs

Using an Identification Number (ID) in CSS rules is very similar to using classes: you assign the ID name to an HTML element then create a style rule for it. The two main differences are:

  • The structure for writing a style rule for an ID is #<idname>.
  • ID names must be unique to the document. Applying the same ID name to more than one HTML element will cause unexpected results in the browser.
ID names are used only once to ensure that a single element can be identified within an HTML document. IDs are often use with JavaScript functions to effect changes on a page.

The code:

    <p>Here is a plain, boring paragraph.</p>
    <p>Here is a another paragraph.</p>
    <p>And another.</p>
    <p id="IDsMustBeUnique">Whoa! what's going on here?</p>

The style rule:

    #IDsMustBeUnique {
      width: 20%;
      float:right;
      background-color: yellow;
      margin: 20px 0;
      padding: 20px;
      border: 4px solid brown;
      font-size: 18px;
    }
