Selectors

You can think of selectors as the nouns of CSS: they tell the browser which element or elements are going to be affected.

Here are some examples of selectors:

<html>
<body>
<h1>
<p>
<ul>

Look familiar? They should: they're the same basic HTML elements you learned about in the HTML section. There are additional ways to identify selectors (classes and IDs), but we'll discuss those later. For now, let's get started with our first CSS style declaration. Let's say I'm going to use the <h1> element for my name on my personal page. I really want it to stand out, so I'm going to add some CSS styling for that tag.

Here's how my <h1> tag looks using only the default style supplied by the browser:

My Name

Create a container for CSS selectors

I need a place to hold my CSS declarations, so I'll add a section inside of the <head> portion of my page, like this:

<head>
<style type="text/css">

</style>
</head>
The 'type="text/css"' section of the style declaration is not needed by browsers using the HTML5 specification. In 2012, that's anyone using a smartphone or the latest versions of Chrome, Firefox Internet Explorer, Opera or Safari. It is, however, required for older browsers using a priors versions of the standard (which is still many, many people). Include it and you will be both backwards-compatible with older browsers and future-compatible in case a successor to CSS is developed.

Basic Selectors

Now I'll tell the browser that I want to change the default style of all <h1> tags. I have to tell the browser when to stop applying our style rules to the <h1> tag and start applying rules fo the next element, so I'll wrap the declaration in braces: {  }

<head>
<style type="text/css">
  h1 {
    color: blue;
  }
</style>
</head>

Note that you don't need to use the < and > symbols. Those are part of the HTML syntax and not necessary here.

I'm also using a standard programming convention for the braces. It's very common to put the opening brace directly after the element, and the closing brace on its own line.

So I've added my h1 selector to my page. Let's see what happens:

My Name

The color changed to blue, as expected. We'll go into more detail on the types of properties you can change in the next section, but let's focus on how to select those items for now.

Sibling Selectors

Let's say we have a document with several paragraphs, but we only want to change the first one. Here's our code:

  <div class="codeExample" id="firstPara">
    <h1>My Name</h1>
    <p>A paragraph about my background.</p>
    <p>A paragraph about my work history.</p>
    <p>Another paragraph.</p>
  </div>

If we wrote our style like this:

  p {
    color:blue;
  }

Then all of the paragraphs would change color.

My Name

A paragraph about my background.

A paragraph about my work history.

Another paragraph.

One way to solve the problem is tell the browser only select the first paragraph. This can be done by telling the browser to look for the p tag that's directly adjacent to the h1 tag, like so:

  h1 + p {
    color:blue;
  }

My Name

A paragraph about my background.

A paragraph about my work history.

Another paragraph.

There are many more ways to identify them, but this is just to give you one example.

Descendant Selectors

You can also specify descendant selectors. This is particularly helpful for lists: if you try to style the parent <ul> or <ol> tags, you won't see any changes.

  ul {
    list-style-type: circle;
  }
  ol {
    list-style: upper-latin;
  }

Our rules weren't applied. That's because the <li> element uses those rules, not the parent element. We need to change our rule to include the descendant element:

  ul li {
    list-style-type: circle;
  }
  ol li {
    list-style: upper-latin;
  }

Now our rules will work.

How to identify selectors

Choosing how to identify selectors to style is confusing at first. A good approach is to try and look past the content of your HTML and see only the elements of your code. Eventually, instead of seeing something like this:

<h3>Digital Services Librarian</h3>
  <p>Illinois Institute of Technology
  <h4>Web Design/Development</h4>
      <ul>
        <li>Creating new digital content for the library: including web-based tutorials, digitally reformatted collections, subject specific pathfinders, and other digital library services. </li>
        <li>Developing dynamic, evolving, <strong>user-centered content</strong> and services for all IIT Library web sites. </li>
        <li>Introducing <strong>best practices</strong> for web development such as version control, usability studies and process documentation.</li>
        <li>Designed and <strong>developed the library's first <a href="http://m.library.iit.edu">mobile website</a></strong> (summer 2011).</li>
      </ul>
