Selectors

You can think of selectors as the nouns of CSS: they tell the browser which element or elements are going to be affected.

Here are some examples of selectors:

<html>
<body>
<h1>
<p>
<ul>

Look familiar? They should: they're the same basic HTML elements you learned about in the HTML section. There are additional ways to identify selectors (classes and IDs), but we'll discuss those later. For now, let's get started with our first CSS style declaration. Let's say I'm going to use the <h1> element for my name on my personal page. I really want it to stand out, so I'm going to add some CSS styling for that tag.

My Name

Add a style section to the page

I need a place to hold your CSS declarations, so I'll add a section inside of the <head> portion of my page, like this:

<head>
<style type="text/css">

</style>
</head>

Then I'll tell the browser that I want to change the default style of all <h1> tags. I have to tell the browser when to stop applying our style rules to the <h1> tag and start applying rules fo the next element, so I'll wrap the declaration in braces: {  }

<head>
<style type="text/css">
  h1 {

  }
</style>
</head>

Note that you don't need to use the < and > symbols. Those are part of the HTML syntax and not necessary here.

I'm also using a standard programming convention for the braces. It's very common to put the opening brace directly after the element, and the closing brace on its own line.

So I've added my h1 selector to my page. Let's see what happens:

Nothing has changed. And that makes sense, right? That because we've identified h1 as the selector we want to change in our style, but we haven't actually made any changes to the style. That's for our next section: Declarations.
