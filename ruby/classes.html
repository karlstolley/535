Show Some Class

Learning Goals

  1. Identify and understand Classes.
  2. Identify and understand Objects.
  3. Understand instantiation.
  4. Understand relationship between Objects and Methods.

Now that you have a script that will create the course list you need, what if you want it to do more later?

That may sound like we are getting ahead of ourselves, but think about it. Right now the script does two things:

  1. Asks questions
  2. Creates the HTML Table for your web page

This functionality is in one script. There's no way to reuse part of it, like the create HTML table part, from another program later. Well, maybe you would not reuse parts of this script, but you should get in the habit of writing modular code: code that has smaller reusable methods that can be reused. That way, you can build off of your previous work.

Sounds nice, right? Well, how would you do that?

You would use Methods. You have been using Methods for a large part of this chapter: .each, .chomp, puts, gets, and others. Where do those Methods come from?

This is where the discussion gets interesting. Ruby is called an Object Oriented Programming Language.

Big words. Simple concept. What can be done in a Ruby program is done by Objects. Objects are parts of a program that have certain properties and can do certain tasks. These properties and tasks (Methods) are unique to that Object. This groups code so you can use an Object to accomplish certain tasks because you will know exactly how that Object would behave.

Wait, if each Object can have its own properties and methods, how can we know those properties and methods?

Objects are not entirely unique. They are built from a blueprint called a Class. OK, let's dive into that concept for a minute:

If you want to build a house, you need a blueprint. This blueprint will tell you that a house has electricity, water, gas, doors, windows and floors. These are common properties of houses. This blueprint will also tell you that a house can get water, get electricity and get gas.

What you do when you program is that you create an Object from a Class. Creating a new instance (copy) of an Object is called instantiation.

To follow the analogy, if we instantiate a House Object, we would say how many floors, doors and windows it has. Houses have basic requirements, but a specific House can have any allowed quantity of doors, windows, etc. because that House is unique.

To create a Class in Ruby, you only need to use the keyword class followed by the name of the Class beginning with a capital letter. This is important to Ruby; it's how it distinguishes between Objects and methods or variables. 
class House

end

The Class is simple. Too simple. We should create a Class with some properties and methods.

class House doors
  number_of_doors = doors
  def open_door
    puts "door is open"
  end
end

That's better. Now, to instantiate an Object, all you need to do is call it:

my_house = House.new(1)

The new House Object my_house now has a property (number_of_doors, set to 1) and a method (.open_door). This would let us write code that would access those capabilities.

my_house.open_door
doors = my_house.number_of_doors

Your script needs to have either the Class and the call to it in the same script or use an include to tell your script which file has the Class file.

Enough theory. Let's get to it.

Class
Blueprint code from which copies of itself can be created with predictable properties and capabilities.
Instantiation
The act of creating an Object using a Class blueprint.
Object
Groups of code in program that have certain properties and can do certain tasks. Objects are instances of a Class.
