That's Some Good Hash

Learning Goals

  1. Identify Arrays.
  2. Identify Hashes.

We hope you're comfortable with variables, because we're going to take you a little further.

OK, a lot further. We need to be able to easily track and work through multiple courses, questions and the like. That will require you to understand how to use placeholders for multiple related items (Arrays, Hashes) and the means to move through and act on those related items (with our old friend the Method). Let's get started.

Variables store one value. That may be all you need. Sometimes, though, you need to keep track of a list of related things. Having a separate variable for each item, like your courses, does not make a lot of sense. Why can't you just have a list of courses and keep track of them? You can, using Arrays.

Arrays

An Array is a list of items. That's it. An Array gives you one identifier to use to work with multiple related items.

An Array is created in the same way as a Variable. It is declared, though it does not need an initial value.

# New Array
courses = []

You can give an Array some values to start:

# New Array
courses = ["com531","com535"]

Arrays are accessed by writing the Array variable and the number of the item in the list you want to use.

So, to show com531 above, you would write:

# Accessing an Array Value
puts courses[1]

Actually, no, you wouldn't. Arrays start counting from 0, as that's what computers do. (They do this because they work in binary systems, not decimal systems. Suffice to say, going along with the computer's convention helps with performance in larger programs.) So, to show com531, you would write:

# Accessing the Right Array Value
puts courses[0]

Well, you may say, this is not the most convenient way to store or recall this information. After all, if the Array is large enough, you wouldn't know what position to request. Well, smart guy (or gal), you would be right. Arrays are great to use when you have to cycle through all items in an Array, but there would have to be an easier way to use a value stored an Array.

There is. It's called an Associative Array, also known as a Hash.

Hashes

The term Hash comes from Perl by way of older computer science concepts. "Hash" is a lot easier to say repeatedly than "associative array". Hash is computer science is short for "Hash Table" which is a type of data that uses identifying values (known as Keys) to find associated values (known simply as values). Whew. Got all that?

So, Ruby uses Hashes to offer a way to store lists of Values that are internally identified by another value (Key) that's not a number. What does that look like?

# Creating a Hash
courses = {}

You can create a populated Hash by writing:

# Creating a Hash
courses = {"com531"=>"Web Development", "com535"=>"Instructional Design"}

So, to get the name of com535 using the Hash, you would write:

# Creating a Hash
puts courses['com535']

Simple, right? Now, we can see you may have two questions:

  1. What's with this operator =>?
  2. Why do we not use curly brackets {} when recalling a value instead of square brackets []?

We have answers for that:

  1. This => is called a Hash Rocket. It is what you use when you assign a Value to a Key in a Hash. It helps you identify that the two values are indeed a Hash and which is the key and which is the value. That's good to know.
  2. The curly brackets {} are used when creating a Hash or assigning values to it. The square brackets [] are used when retriving values, like an Array, because a Hash is an Array. The only difference is that the key in an Array can be something other than a number.

Did you get all that? Good. Now, let's get into why Hashes can make your code easier to read and use.

Array
A list of items using one identifier to work with multiple related items.
Hash
An Associative Array that stores lists of Values internally identified by another value called a Key rather than a number.
Hash Rocket
An Operator (=>) which links keys and values in an Associative Array in Ruby.
