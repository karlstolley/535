Values, Expressions, and Functions

We're going to talk more about working with the REPL soon, but before we do that, let's talk about what a program is. Afterall, JavaScript is a programming language.

For our purposes, let's say that a program consists of a set of stuff, and a list of instructions to perform on that stuff to produce new stuff. Admittedly, this is a bit a vague. But let's use a calculator as an example.

When you type 2+3 into your calculator, that is a program. The stuff I was referring to before are the numbers 2 and 3. They are numbers -- one of the most basic things your calculator can understand. We are going to refer to these as values. In between these values, you type +. This is an instruction to the calculator. What you are saying is, "I want you to take these two numbers, and add them together." We'll refer to an instruction as a function. When you apply the addition function to the values 2 and 3, you get a result: 5.

You could type something more complex into your calculator. For example, you could type:
(10 * 5) / 25
In this case, we have a couple of instructions we are giving the calculator: "First, multiply 10 by 5, and then divide the result by 25." To use the terms we just introduced, we want to apply the multiplication function to the values 10 and 5. That will give us a result -- another value (50). Then we want to apply the division function to the values 50 and 25. Our final result is 2. What's different in this example is that we have to apply one function before we can apply the other -- we have to know what 10 * 5 is before we can divide it by 25. To introduce one more term, an expression is anything that produces a value. (10 * 5) is an expression because it produces the value 50. (10 * 5) / 25 is an expression because it produces the value 2. An expression consists of some values and the functions you are applying to them. A program is just a big list of expressions.

Let's get some practice with these things inside of our JavaScript REPL. I've been referring to this thing we've been looking for as a REPL, but what does that mean? It stands for Read Eval Print Loop. At first, this might look like jargon, but let's break it down.

When you type something into the REPL, first it reads it. That just means it looks at what you typed in. Then it evaluates it. We'll get into this in just a second, but what this basically means is it applies some rules to what you typed in. Finally, it prints the results. Loop just means it never stops -- after it prints out the result, it will be ready to read in the next thing you have to say.

Open up your REPL and type in:
42
What happened? It should just print the number 42 right back at you. I said before that evaluation means applying some rules to what you typed in. 42 in JavaScript, like in our calculator example, is a value. The rule for values in the REPL is that they evaluate to themselves.

Now type in:
42 + 3
What happened? It should print the number 45. Instead of typing in a value, you typed in an epression. The rule the REPL follows for expressions is to apply the functions in the expression to the values, resulting in a single value. If you type in 42 + 3 + 7 + 3, it will apply the add function to all those values, and it will print the result 55.

So far we've just been using the REPL as a calculator, giving it numeric values. But there are other types of values in JavaScript too. One of them is a string. Try typing this into the REPL:
"Hello!"
"Hello!" is a string value. You can tell it's a string because it is surrounded by quotation marks. We'll look more at using strings later.

Now try typing:
"Hello!".toUpperCase()
What happened? It should print out "HELLO!" Why did it do that? What we just typed in was an expression. It looks different than 2+3, but it actually has all the same elements. In this case, we have the string value "Hello!". .toUpperCase() is actually a function that we are asking the REPL to apply to that value.

Finally, try typing:
alert("Hello!")
What happened? You should have gotten a pop up window from your browser that said "Hello!" In this case, alert is a function, and we're giving it the value "Hello!". But what does the REPL say? It should print out something like undefined. Up until now, we have been typing expressions into the REPL, and we have only looked at functions that produce values. Sometimes, we call functions that don't produce values, and instead we call them for their side effects. In this case, the side effect of calling alert is that the browser pops open a little window with our message, but the alert function does not produce a new value.
