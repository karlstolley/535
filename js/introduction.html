JavaScript

An Introduction to JavaScript

To get an idea of what JavaScript is, it's useful to think of some of the things we can do with it.

  • When Google tries to guess what you're typing in its search box and displays different possibilities, that's JavaScript.
  • When you're signing up for a new account on a website, and it tells you in real time as you type your password if it's strong enough, that's JavaScript.
  • When you you click on an item on a menu, and it expands to show you more options, that's JavaScript.

What do these things have in common? They all have to do the behavior of a webpage. JavaScript is a programming language that is implemented in all modern web browsers that let us write programs to allow for greater interactivity on a webpage. While HTML lets you define the content of a page, and CSS lets you define what that content looks like, JavaScript lets you define how it behaves and how it should interact with whoever is looking at it.

Learning the basics of JavaScript will allow you to add this kind of behavior to your web content. In addition to letting you make more interesting webpages, JavaScript is a great programming language to learn because of its ubiquity. As mentioned before, almost all modern web browsers include a JavaScript interpreter, so knowing JavaScript lets you write scripts that can run on almost any platform.

A history of JavaScript

It is useful to know a few things about the history of JavaScript, to get familiar with some of the terms you'll hear as you read more about it.

JavaScript was invented by a guy named Brendan Eich for Netscape back in the 1990's. Netscape was the maker of a web browser, so at that time, they made a competitor product to Microsoft's Internet Explorer. It was originally called Mocha and LiveScript, and the name JavaScript was finally chosen mostly for marketing reasons, since the Java programming language was becoming popular at that time.

Microsoft wanted to be able to support similar functionality in Internet Explorer, so they added a language called JScript to their browser. JScript was similar to JavaScript, but not necessarily completely compatible. By the mid 90's, it was clear that having a scripting language in the browser was important, but that it would be too much of a hassle to have to support different implementations in different browsers. So a standard was written for the language. The standards body resposnible for this standard is called ECMA, so the standard for the language is called ECMAScript.

So, what can we learn from this history?

  • JavaScript has almost nothing to do with Java, besides sharing a name. So if you don't know Java, don't think you're behind in learning JavaScript. If you do know Java, you might be a little more comfortable reading some JavaScript, but you shouldn't assume things work the same -- these are two different languages.
  • There is a standard for JavaScript, but each browser still has their own JavaScript engine that implements that standard. Mozilla's JavaScript engine is called SpiderMonkey. Google's JavaScript engine is called V8. This shouldn't matter for this tutorial, but if you run into these names when you are reading about JavaScript, they just refer to different browsers.
  • Although there is a standard, certain browsers might not follow the standard completely. Everything we'll do in this tutorial should work in all modern web browsers, and for the most part, most JavaScript scripts that follow the standard should also work. But it is worth keeping in mind that there are still differences between browsers, so as you read more about JavaScript on the web, you might notice some people being annoyed with with these differences.
  • ECMAScript refers to the standard, and JavaScript refers to browsers' implementation of that standard. The latest edition of ECMAScript is 5.1, and most modern web browsers will support that standard. We won't be doing anything too fancy that was introduced in the newest version, but it's important to know that if you're interested in the direction that JavaScript is heading, you should see what proposals are being made for the next version of ECMAScript.
